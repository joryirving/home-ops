## Task: Analyze cronjobs showing error status

### Root Cause Identified

The cronjobs are failing due to **channel target mismatches**:

1. **Invalid Discord Channel IDs**: Cronjobs trying to send to Discord using:
   - `-1002381931352` (Telegram channel ID format)
   - `miso-random-selfie` (channel name, not ID)
   - `miso` (channel name, not ID)

2. **Valid Discord Channel**: `1466128096125255750` (configured in openclaw.json)

3. **Root Issue**: The gh-issues skill's `--notify-channel -1002381931352` is documented as a **Telegram** channel ID, but crons are invoking it without specifying `--channel telegram`, causing Discord to fail.

### Evidence from Logs

```
[tools] message failed: Unknown target "-1002381931352" for Discord.
[tools] message failed: Unknown target "miso-random-selfie" for Discord.
[tools] message failed: Unknown Guild
```

### Proposed Fixes

1. **For Telegram notifications**: Add `--channel telegram` flag when using Telegram channel IDs
2. **For Discord**: Use channel ID `1466128096125255750` (not names)
3. **Alternative**: Update HEARTBEAT.md to use the correct Discord channel ID instead of Telegram

### Secondary Issues

- **Memory quota exceeded**: OpenAI 429 insufficient_quota errors (separate issue)
- **File path mismatch**: `memory/proactive-tracker.md` vs `proactive-tracker.md`
